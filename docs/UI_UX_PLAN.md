# UI/UX Improvement Plan

## Current pain points
- New users have limited onboarding context after opening the app.
- Important actions are discoverable but not strongly guided.
- Dense layout can feel busy when multiple panes are open.

## Goals
1. Improve first-run clarity.
2. Reduce interaction friction for frequent actions.
3. Increase visual hierarchy and consistency.

## Action plan
### Phase 1 (quick wins)
- Improve README + in-app guidance references.
- Ensure empty states explain next actions clearly.
- Add clearer labels/tooltips for panel toggle buttons.

### Phase 2 (interaction quality)
- Add command palette for file actions and navigation.
- Add contextual right-click menu for file tree items.
- Improve keyboard focus visibility across panes/modals.

### Phase 3 (visual polish)
- Define spacing and typography scale tokens.
- Normalize iconography and status colors.
- Review contrast and accessibility across themes.

## Success metrics
- Reduced time-to-first-note for new users.
- Fewer misclicks around panel toggles and file operations.
- Positive qualitative feedback on layout clarity.
